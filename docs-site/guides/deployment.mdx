## VPS Deployment

### 1. Install

```bash
curl -fsSL https://overseer.sh/install | bash
```

### 2. Configure Nginx (Optional)

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### 3. SSL with Certbot

```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

### 4. Systemd Services

```bash
sudo cp systemd/*.service /etc/systemd/system/
sudo systemctl enable --now overseer
sudo systemctl enable --now overseer-discord
```

## Docker Deployment

```bash
docker compose up -d
```

## Deploying the Documentation (docs-site)

The Overseer documentation (`docs-site/`) is built using [Mintlify](https://mintlify.com) and can be easily deployed to Vercel or hosted on Mintlify's platform.

### Local Testing

To test the documentation locally before deploying, run the Mintlify CLI:

```bash
cd docs-site
npx mintlify dev
```

The documentation will be available at `http://localhost:3000`.

### Vercel Deployment

To deploy the documentation site to Vercel:

1. Push your code to a GitHub repository.
2. Go to the [Vercel Dashboard](https://vercel.com/dashboard) and click **Add New...** -> **Project**.
3. Import your Overseer repository.
4. **Important Config:**
   - **Framework Preset**: Other
   - **Root Directory**: `docs-site`
   - **Build Command**: `npm run build` (or leave default if Mintlify handles it, but typically Mintlify docs are deployed via Mintlify's own GitHub app. If deploying a custom static export, ensure Mintlify export commands are used).
   
*Note: The easiest way to host Mintlify docs is often through the Mintlify dashboard by linking your GitHub repository directly to Mintlify.*
